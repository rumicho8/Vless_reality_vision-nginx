# Xray Reality Ultimate Integration (Stability Edition)

本仓库提供了一套针对 Linux 服务器深度优化的 VLESS-XTLS-Vision-REALITY 自动化部署方案。

## 🖥️ 系统环境要求 (System Requirements)

### 1. 操作系统
- **Debian (推荐)**: 11, 12, 13 (全版本自适应适配)
- **Ubuntu**: 20.04, 22.04, 24.04
- *注意*: 脚本针对 Debian 13 的 `/etc/sysctl.d/` 路径进行了专项逻辑适配。

### 2. 硬件资源
- **内存 (RAM)**: 最低 512MB (脚本已内置 100MB 日志配额限制以节省资源)
- **架构**: 支持 amd64 (x86_64) 和 arm64-v8a (ARM 架构)

### 3. 网络环境
- **入站端口**: 必须放行 80 (HTTP) 和 443 (或自定义的 Xray 监听端口)
- **外部依赖**: 能够访问 GitHub (用于下载 Xray 核心和伪装网页)

## 🛠️ 脚本详细功能拆解

### A. 系统层优化
- **BBR 加速**: 自动识别内核并开启 TCP BBR + FQ，优化高延迟网络下的吞吐。
- **日志管理**: 自动配置 `systemd-journald`，防止日志无限增长撑爆磁盘。
- **进程锁机制**: 自动清理 `apt` 进程锁，解决软件安装时的冲突问题。

### B. 安全防御体系
- **SNI 伪装**: 提供“Web 回落”与“SNI 偷取”双模式。
- **Nginx 防御探针**:
    - 针对 Nginx >= 1.22.0 开启 `ssl_reject_handshake`，令未携带合法 SNI 的探测请求直接被 TCP 重置。
    - 预设 HSTS、X-Frame-Options 等安全头，隐藏服务器指纹。
- **隐私保护**: 强制 ACME 证书工具在无日志模式下运行。

### C. 自动化控制
- **Systemd Timer**: 使用高精度定时器替代 Cron 任务。
- **资源热更新**: 每周自动拉取最新的 `geoip.dat` 与 `geosite.dat` 规则，实现广告拦截与中国 IP 绕路。

## 📂 脚本版本对比 (1.sh vs 2.sh)

| 功能模块 | `1.sh` (标准宽泛版) | `2.sh` (严格单域版) |
| :--- | :--- | :--- |
| **域名预处理** | 自动过滤 `www.` 前缀，提取主域名 | 严格使用用户输入的内容 |
| **证书申请** | 同时申请 `domain.com` 和 `www.domain.com` | 仅申请 `domain.com` |
| **通配符支持** | DNS 模式下支持 `*.domain.com` 泛域名 | 不支持泛域名 |
| **Nginx 路由** | 同时响应主域与 www 子域的 HTTPS 请求 | 仅响应主域请求 |
| **Reality SNI** | 允许 `www` 和主域作为 Reality 握手域名 | 严格限制为单一主域名 |

## 🚀 部署流程

1. **获取脚本**: 将代码保存为 `1.sh` 或 `2.sh`。
2. **赋予权限**: `chmod +x 1.sh`
3. **运行脚本**: `./1.sh`
4. **选择模式**:
    - **模式 1 (推荐)**: 适合拥有域名且希望自建伪装站的用户。
    - **模式 2**: 适合无域名，需借用 Apple/Microsoft 等大厂域名的用户。

## 🧹 卸载与清理

运行脚本并选择选项 `2` 即可进入卸载菜单。
- 脚本支持**焦土式卸载**：可连带删除 Nginx、Socat、JQ 等系统依赖包，恢复纯净环境。
- 自动保留 BBR 优化与日志限制策略，确保系统重启后的基础性能。
